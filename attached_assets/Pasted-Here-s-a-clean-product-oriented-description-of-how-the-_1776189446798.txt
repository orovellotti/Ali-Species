# ⚙️ 4. How to compute it (implementation logic)

## Step 1 — Extract statuses from existing data

You already have:

* red_list
* protection
* directives
* conventions
* regulation
* pna
* znieff

---

## Step 2 — Normalize each status

Map to values (0–1):

Example:

```js
redListScore = {
  CR: 1.0,
  EN: 0.8,
  VU: 0.6,
  NT: 0.4,
  LC: 0.2,
  DD: 0.3
}
```

Same idea for:

* protection
* directives
* etc.

---

## Step 3 — Compute dimension scores

```js
ecological = redListScore

regulatory = max(
  protectionScore,
  directiveScore,
  conventionScore
)

territorial = average(
  znieffScore,
  pnaScore
)

management = invasiveScore
```

👉 Use:

* **max** for legal (strongest constraint wins)
* **average** for contextual signals

---

## Step 4 — Aggregate global score

```js
global =
0.4 * ecological +
0.3 * regulatory +
0.2 * territorial +
0.1 * management
```

Then:

```js
score100 = Math.round(global * 100)
```

---

## Step 5 — Generate explanation

Rule-based:

```js
if (redList === "EN")
  explanation.push("Endangered status increases ecological sensitivity")

if (protection === "national")
  explanation.push("National protection increases regulatory sensitivity")
```

👉 Keep it simple and deterministic

---

## 🧠 5. Smart UX behaviors

### Highlight inconsistencies (very valuable)

Example:

* High ecological + low regulatory → show:

  > “High ecological risk with limited legal protection”

---

### Handle missing data

* If no Red List:
  → reduce ecological confidence
  → show: “Limited ecological data”

---

### Avoid duplication

You already show statuses →
👉 the score should **summarize, not repeat**
